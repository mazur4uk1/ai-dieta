# Изменения в AI-Dieta: Переход с локальной нейросети на онлайн API

## Проблема
Исходное приложение использовало локальную нейросетевую модель YOLOv8 (`yolov8n.pt`) для распознавания еды, что приводило к:
- Большому размеру Docker образа (более 12ГБ)
- Высоким требованиям к ресурсам
- Сложности в поддержке и обновлении моделей

## Решение
Замена локальной нейросетевой модели на бесплатные онлайн API для распознавания еды.

## Внесенные изменения

### 1. Обновление vision.py API
- **Удалено**: Зависимости от `ultralytics` и `opencv-python-headless`
- **Добавлено**: Интеграция с Edamam Food Detection API
- **Добавлено**: Поддержка альтернативных API (Google Cloud Vision, Azure Computer Vision, Clarifai)
- **Добавлено**: Fallback-режим для случаев отсутствия API-ключей
- **Добавлено**: Эндпоинт `/available-apis` для получения информации о доступных API

### 2. Обновление зависимостей (requirements.txt)
- Удалены: `ultralytics==8.0.134`, `opencv-python-headless==4.8.1.78`
- Оставлены: `httpx==0.25.2` (для HTTP-запросов к API)

### 3. Обновление Docker конфигурации
- **.dockerignore**: Добавлено исключение `yolov8n.pt` для уменьшения размера образа
- **Dockerfile**: Без изменений (уже оптимизирован)

### 4. Документация
- **FOOD_RECOGNITION_SETUP.md**: Подробная инструкция по настройке API
- **README.md**: Обновлено описание функциональности
- **test_vision_api.py**: Тестовый скрипт для проверки API

## Преимущества изменений

### 1. Размер Docker образа
- **До**: >12ГБ (из-за локальной модели и зависимостей)
- **После**: ~100-200МБ (только зависимости FastAPI)

### 2. Точность распознавания
- **До**: Ограниченная модель YOLOv8 (не специализированная на еде)
- **После**: Профессиональные API, обученные на больших датасетах еды

### 3. Поддержка и масштабируемость
- **До**: Необходимость обновления локальных моделей
- **После**: API обновляются провайдерами автоматически

### 4. Ресурсы
- **До**: Требуется GPU/CPU для inference
- **После**: Только интернет-соединение

## Настройка API

### Edamam Food Detection API (рекомендуется)
1. Зарегистрируйтесь: https://developer.edamam.com/
2. Получите APP_ID и APP_KEY
3. Установите переменные окружения:
   ```bash
   export EDAMAM_APP_ID=ваш_app_id
   export EDAMAM_APP_KEY=ваш_app_key
   ```

### Альтернативные API
- **Google Cloud Vision**: 1000 единиц/месяц бесплатно
- **Azure Computer Vision**: 5000 транзакций/месяц бесплатно
- **Clarifai**: 5000 предсказаний/месяц бесплатно

## Использование

### API Эндпоинты
- `POST /api/vision/recognize` - Распознать еду на изображении
- `GET /api/vision/available-apis` - Получить информацию о доступных API

### Пример запроса
```bash
curl -X POST "http://localhost:8000/api/vision/recognize" \
  -H "Authorization: Bearer ваш_токен" \
  -F "image=@путь/к/изображению.jpg"
```

### Пример ответа
```json
{
  "recognized_foods": [
    {
      "name": "Apple",
      "calories": 95,
      "confidence": 0.9
    }
  ]
}
```

## Fallback-режим
Если API не настроены или недоступны, приложение автоматически переключается на fallback-режим, возвращая:
- Список常见ых продуктов
- Приблизительную калорийность
- Уверенность распознавания

## Тестирование
Запустите тестовый скрипт:
```bash
cd backend
python3 test_vision_api.py
```
